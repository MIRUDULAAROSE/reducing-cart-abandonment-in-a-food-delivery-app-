# reducing-cart-abandonment-in-a-food-delivery-app-
Reducing cart abandonment through different analysis and proposing solution to overcome the problem and boost revenue 

# Problem Statement:
         Users are adding items to the cart but not completing the purchase due to factors such as high delivery rate (especially during late hours), limited
payment options, app lag during checkouts and a complex checkout process. This leads to increase cart abandonment resulting in reduced order completion rates 
and potential revenue loss for the business.

# Objective:
        The objective of this project is to improve cart abandonment by analysing key factors affecting the checkout process and improvimg user experience to 
increase conversion rates.

# Approach:
        I approached this problem by analyzing key areas including user behaviour before cart abandonment, the checkout process, payment experience and customer 
feedback. This helped identify friction points such as drop-offs during checkouts, payment issues and usability concerns. I also reviewed competitor offering 
and pricing to understand external factors influencing user decisons.

# Analysis:
        1. User Behaviour Analysis: I analyzed the user journey to understand the steps users take before abandoning the cart, identifying the stage where maximum 
drop-offs occur. I also reviewed metrics such as conversion rates, number of users proceeding to checkout vs. completing orders, and patterns in items added to cart.
Additionally, I examined user demographics and geographical data to identify if the issue is region-specific.
        2. Checkout / Payment Analysis: I evaluated the checkout and payment process by analyzing factors such as page load time, payment failures, available payment methods, 
and the number of steps in the checkout flow. I also reviewed application performance metrics across Android and iOS platforms to identify any technical issues affecting the
final transaction stage.
        3. Customer Feedback Analysis: I analyzed customer feedback by reviewing user reviews, ratings, and complaints related to checkout and payment issues. This included 
identifying concerns about high delivery charges, limited payment options, and overall dissatisfaction with the final ordering experience.

# Root Cause Identification:
        The primary root cause identified is an inefficient checkout and payment process, characterized by slow page load times, occasional crashes during checkout, and 
limited payment options. These issues disrupt the final transaction stage, leading to poor user experience, reduced customer satisfaction, and increased cart abandonment.

# Proposed Solution:
        The proposed solution focuses on reducing friction at the final stage of the user journey by optimizing the checkout and payment process. This includes improving 
page load time, eliminating crashes, simplifying the checkout flow, and expanding available payment options to ensure a smooth transaction experience. 
Additionally, targeted offers, reward coupons, and communication campaigns can be introduced to rebuild customer trust and encourage users to complete their purchases.

# Primary User Story:
        As a food delivery app user, I want a fast and smooth checkout and payment process, so that I can complete my order without delays or frustration.

# Supporting User Story:
        1. As a user, I want multiple payment options, so that I can choose a convenient method to complete my order.
        2. As a user, I want access to support during checkout, so that I can resolve any issues quickly.
        3. As a user, I want to receive relevant offers and discounts, so that I feel encouraged to complete my purchase.

# Acceptance Criteria:
        1. Checkout and payment pages should load within 2 seconds under normal network conditions.
        2. Users should be able to complete the checkout process in a maximum of 3 steps.
        3. Payment process should be able to completed in a maximum of 3 steps without errors.
        4. At least 3 payment options (e.g., UPI, card, wallet) should be available and functional.
        5. The system should successfully process payment without crashes or transaction failures.
        6. Users should receive an order confirmation immediately after successful payment.
